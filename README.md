# Structure Is Not Function: Alignment Algorithms on Bacterial Proteins

CMU Pre-College Computational Biology — Final Project

## What this is

Three pairwise alignment algorithms (Needleman-Wunsch, Smith-Waterman, and a
simplified BLAST-style seed-and-extend heuristic), implemented from scratch
in Python, tested on three classes of bacterial proteins:

- **β-lactamase** (target — antibiotic resistance enzyme)
- **tRNase Z** (its likely structural ancestor — same fold, different job)
- **Transcription factors** (negative control — unrelated fold and function)

Research question: do these algorithms distinguish proteins by *function*, or
only by structural/evolutionary similarity?

## Project structure

```
data/sequences.fasta      the protein database (FASTA format)
scripts/bio_utils.py      FASTA parser + BLOSUM62 substitution matrix
scripts/needleman_wunsch.py   global alignment
scripts/smith_waterman.py     local alignment
scripts/mini_blast.py         seed-and-extend heuristic
scripts/run_analysis.py       runs all three across the whole database
website/index.html         the final deliverable (self-contained, no build step)
website/results.json       generated by run_analysis.py; already embedded in index.html
```

## IMPORTANT — before you submit anything

`data/sequences.fasta` currently contains **two real sequences** (E. coli
TEM-1 β-lactamase, and a Pseudomonas putida RNase Z) and **several
PLACEHOLDER entries**. You must replace every PLACEHOLDER with a real
sequence for your three chosen species before treating any result as a
finding. This is intentional — picking the species and pulling real data is
part of the checkpoint requirements, not something to skip.

To get real sequences:
1. Go to https://www.uniprot.org
2. Search e.g. `gene:blaSHV AND organism:"Klebsiella pneumoniae"`
3. Open the entry → Format → FASTA, copy the sequence
4. Paste it into `data/sequences.fasta`, following the existing header format:
   `>CLASS|Species_name|gene|ACCESSION description`

## How to run it

```bash
cd scripts
python3 run_analysis.py                    # skips PLACEHOLDER entries
python3 run_analysis.py --include-placeholders   # runs on everything, for testing
```

This writes `website/results.json`. To get the website working with your new
data, re-embed the JSON into the HTML:

```bash
cd website
python3 -c "
import json
data = open('results.json').read()
html = open('index.html').read()
import re
html = re.sub(r'(<script id=\"resultsData\" type=\"application/json\">).*?(</script>)',
              lambda m: m.group(1) + data + m.group(2), html, flags=re.S)
open('index.html', 'w').write(html)
"
```

(Or just open `index.html`, find the `<script id="resultsData">` block, and
paste the new JSON in by hand — same result.)

## Viewing the site

`index.html` is self-contained (the data is embedded directly, no server
needed) — just open it in a browser. For GitHub Pages: push the `website/`
folder's contents to your repo and enable Pages on that folder/branch.
